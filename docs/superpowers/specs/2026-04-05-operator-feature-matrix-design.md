# Operator Feature Matrix

Auto-generated Markdown table tracking which PyTorch features each operator supports.

## Purpose

Provide a single-glance view of implementation coverage across all operators and all PyTorch integration features. The table is generated by a Python script that introspects the built library at runtime, so it reflects ground truth.

## Output

A Markdown file at `docs/operator-feature-matrix.md` containing five tables (one per feature group), each with a summary row. All tables share the same operator rows, sorted alphabetically.

### Rows

All forward operators registered with the `torchscience` namespace. Discovered at runtime by filtering `torch._C._dispatch_get_registrations_for_dispatch_key('CPU')` for entries starting with `torchscience::` and excluding any containing `_backward`.

## Script

A Python script at `scripts/generate_operator_feature_matrix.py`.

### Detection Strategy

The script uses two PyTorch internal APIs for dispatch key detection:

```python
# Get all operator names registered under a dispatch key
torch._C._dispatch_get_registrations_for_dispatch_key(key)
# Returns list of strings like "torchscience::gamma", "torchscience::gamma_backward", etc.

# Check if a specific operator has a kernel for a dispatch key
torch._C._dispatch_has_kernel_for_dispatch_key(op_name, key)
# Returns bool
```

For dtype detection, the script constructs test inputs and attempts to call the operator:

```python
schema = torch.ops.torchscience.<op>.default._schema

# Count tensor arguments (excluding non-tensor params like int64_t)
tensor_args = [a for a in schema.arguments if str(a.type) == "Tensor"]

# Build test inputs of the target dtype
inputs = [torch.ones(1, dtype=target_dtype) for _ in tensor_args]

# Attempt the call
try:
    torch.ops.torchscience.<op>(*inputs)
    supported = True
except (RuntimeError, NotImplementedError):
    supported = False
```

For operators with non-tensor parameters (e.g., `log_multivariate_gamma` takes `int64_t d`), the script provides a default value (e.g., `1` for int parameters).

### Operator Discovery

```python
import torch
import torchscience.special_functions  # triggers library load

all_regs = torch._C._dispatch_get_registrations_for_dispatch_key('CPU')
ts_ops = [r for r in all_regs if r.startswith('torchscience::')]
forward_ops = sorted(set(
    op.replace('torchscience::', '')
    for op in ts_ops
    if '_backward' not in op
))
```

### Backend Detection

For each forward operator and each dispatch key:

```python
DISPATCH_KEYS = {
    "CPU": "CPU",
    "CUDA": "CUDA",
    "Meta": "Meta",
    "Autograd": "Autograd",
    "AutocastCPU": "AutocastCPU",
    "AutocastCUDA": "AutocastCUDA",
    "SparseCPU": "SparseCPU",
    "SparseCUDA": "SparseCUDA",
    "SparseCsrCPU": "SparseCsrCPU",
    "SparseCsrCUDA": "SparseCsrCUDA",
    "QuantizedCPU": "QuantizedCPU",
    "QuantizedCUDA": "QuantizedCUDA",
    "NestedTensorCPU": "NestedTensorCPU",
    "NestedTensorCUDA": "NestedTensorCUDA",
}

for op in forward_ops:
    for col, key in DISPATCH_KEYS.items():
        has = torch._C._dispatch_has_kernel_for_dispatch_key(
            f'torchscience::{op}', key
        )
```

### Hessian Detection

```python
all_cpu = set(
    r.replace('torchscience::', '')
    for r in torch._C._dispatch_get_registrations_for_dispatch_key('CPU')
    if r.startswith('torchscience::')

for op in forward_ops:
    has_hessian = f'{op}_backward_backward' in all_cpu
```

## Usage

```bash
uv run python scripts/generate_operator_feature_matrix.py
```

Requires a built `torchscience` library (i.e., `uv sync` must have been run). Uses only the standard library plus `torch` and `torchscience`.
